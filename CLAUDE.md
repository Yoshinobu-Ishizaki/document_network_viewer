## document_network_viewer

A locally-run Python web app for browsing a collection of documents (PDF and Markdown) via an interactive drill-down graph network.

### Core Features

- **Graph view**: Interactive network graph showing documents organized into a 3-level hierarchy.
- **Drill-down navigation**: Click a group node to expand/zoom into its children. Back-navigation to parent is supported.
- **Document view**: Clicking a document node (leaf) opens its content in a side panel (split-view) alongside the graph.
- **Pre-processing pipeline**: A separate CLI script categorizes documents using the Claude API and writes a cached index.

### Architecture

#### Backend: FastAPI
- Serves the REST API and static frontend files.
- Endpoints:
  - `GET /api/graph` — returns the full graph structure (JSON)
  - `GET /api/doc/{filename}` — returns rendered document content (Markdown as HTML, or PDF metadata)
- Reads from `data/index.json` (generated by the preprocessing script). Never reads raw files from `data/` at runtime.

#### Frontend: Static HTML + JavaScript
- Single HTML page served by FastAPI.
- Uses **vis.js Network** (or D3.js) for the interactive graph.
- Split-view layout: graph on the left, document panel on the right.
- Drill-down behavior: clicking a group node filters the graph to show only that group's children.

#### Preprocessing Script: `preprocess.py`
- Run once with: `uv run preprocess.py`
- Scans `data/` for `.md` and `.pdf` files.
- Calls the Claude API to categorize each document:
  - **Level 1 (top)**: User-defined categories, specified in `config.yaml`.
  - **Level 2 (middle)**: LLM-assigned subcategories within each L1 group.
  - **Level 3 (leaf)**: Individual documents.
- Writes the result to `data/index.json` (graph-ready JSON structure).
- Re-running the script regenerates `data/index.json` (idempotent, with caching to avoid redundant API calls).

### Configuration: `config.yaml`
- User defines the top-level (L1) category names.
- Example:
  ```yaml
  categories:
    - Software
    - Manufacturing Parts
    - Tools & Equipment
  ```

### Data

- Do NOT read raw files in the `data/` folder at app runtime. The app only reads `data/index.json`.
- The preprocessing script (`preprocess.py`) is the only component that reads raw files from `data/`.
- Do not assume or rely on the content of documents in `data/` when writing code.

### Python

- Use `uv` to run scripts and add packages: `uv run <script>`, `uv add <package>`.
- Python >= 3.13.
- Use the `anthropic` SDK for Claude API calls in `preprocess.py`.

### Project Structure

```
document_network_viewer/
├── app.py              # FastAPI app entry point
├── preprocess.py       # Preprocessing CLI script
├── config.yaml         # User-defined L1 categories
├── pyproject.toml
├── data/
│   ├── index.json      # Generated by preprocess.py (gitignored)
│   └── *.md / *.pdf    # Source documents (do not read at runtime)
└── static/
    ├── index.html
    ├── main.js
    └── style.css
```

### Running the App

```bash
# Step 1: Preprocess documents (generates data/index.json)
uv run preprocess.py

# Step 2: Start the web server
uv run app.py
# App available at http://localhost:8000
```
